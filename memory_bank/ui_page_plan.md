# HSO 用户界面页面规划

## 目的

本文档是 HSO 基础产品第 4 步的唯一权威来源。

冻结内容：

1. 完成基础产品闭环所需的最小页面与区域。
2. 每个页面或区域的单一职责与明确禁止项。
3. 第一版必须具备的空态、加载态、成功态和错误态。

本文档**不**定义研究输入类型、检索优先级、研究卡片字段契约或项目创建载荷。这些内容属于第 5 步及之后的流程规划。

## 1. 最小页面与区域列表

第一版只保留三个顶层页面，不再为模板、资产、构建或 PDF 预览拆出额外顶层路由。

### 顶层页面

1. `research_input_page`
   基础产品入口页。用户在这里发起研究输入，并进入研究卡片流程。

2. `research_card_results_page`
   研究卡片结果页。用户在这里审阅研究结果，并触发“转化为论文项目”。

3. `paper_project_detail_page`
   论文项目主工作页。用户在这里围绕单个 `paper_project` 完成模板选择、结构编辑、素材管理、构建状态查看和 PDF 预览/导出。

### `paper_project_detail_page` 内的必需区域

以下四个区域是 `paper_project_detail_page` 的受控子区域，而不是独立页面：

1. `template_selection_panel`
   用于查看当前模板、切换支持目录中的模板，以及显示模板映射不匹配提示。

2. `asset_panel`
   用于查看、添加和移除当前项目需要的受控构建素材。

3. `build_status_panel`
   用于触发构建、查看最新 `build_job` 状态、阅读构建摘要，以及查看可读错误信息。

4. `pdf_preview_export_panel`
   用于展示最近一次成功构建的 PDF 预览，并提供“打开”和“导出”操作。

## 2. 用户旅程与页面边界

基础产品的最小 UI 旅程固定为：

1. 用户进入 `research_input_page`。
2. 提交研究输入后进入 `research_card_results_page`。
3. 在研究卡片确认“转化为论文项目”后进入 `paper_project_detail_page`。
4. 在 `paper_project_detail_page` 内完成模板、结构、资产、构建和预览闭环。

因此第一版不需要以下额外顶层页面：

1. 独立模板中心页。
2. 独立资产库页。
3. 独立构建历史页。
4. 独立 PDF 预览页。
5. 与基础产品闭环无关的全局 dashboard。

## 3. 页面与区域职责

### `research_input_page`

主要职责：
承载研究起点输入、最近一次输入恢复，以及进入研究卡片流程的明确提交动作。

不应做什么：
不直接展示研究卡片细节，不承担项目创建，不承载检索编排逻辑或结果归因逻辑。

### `research_card_results_page`

主要职责：
展示单张 `research_card` 的结构化结果，并提供重试、返回修改输入和转化为项目的操作入口。

不应做什么：
不承担模板选择，不直接编辑 `paper_project` 章节，不在页面内执行项目创建编排细节。

### `paper_project_detail_page`

主要职责：
围绕单个 `paper_project` 汇总主工作流，让用户在一个连续上下文内完成结构编辑、模板绑定、素材管理、构建查看和 PDF 操作。

不应做什么：
不直接执行 LaTeX 构建，不直接操作数据库，不在 UI 层承载模板映射规则或错误归类逻辑。

### `template_selection_panel`

主要职责：
展示当前模板状态、可选模板列表，以及模板切换后的映射反馈。

不应做什么：
不保存模板包实现细节，不自行决定章节映射规则，不承担模板兼容性推理。

### `asset_panel`

主要职责：
展示当前项目已附加的构建相关素材，并提供受控的添加、替换和移除入口。

不应做什么：
不变成通用文件管理器，不承担跨项目素材权限计算，不直接操作底层存储路径。

### `build_status_panel`

主要职责：
展示构建状态流转、最近一次构建摘要、错误摘要与修复建议，并提供重新构建入口。

不应做什么：
不直接运行 `latexmk` 流程，不解析原始构建日志为领域对象，不维护 worker 生命周期。

### `pdf_preview_export_panel`

主要职责：
展示最近一次成功构建的 PDF，并提供内嵌预览、系统打开和导出动作。

不应做什么：
不负责决定哪次构建算“成功结果”，不承担构建失败诊断，不在没有成功产物时伪造预览内容。

## 4. 页面与区域状态变体

第一版要求每个页面或区域都能显式表达空态、加载态、成功态和错误态。若某状态不对应真实数据对象，也必须通过可见反馈说明当前不可操作原因。

### `research_input_page`

1. 空态
   尚未提交输入时，显示研究起点输入表单和输入说明。
2. 加载态
   提交后等待研究卡片生成结果时，禁用重复提交并显示进度反馈。
3. 成功态
   最近一次输入已被接受，页面可跳转或恢复到对应结果页。
4. 错误态
   研究生成失败时显示可见错误、保留原输入，并提供重试入口。

### `research_card_results_page`

1. 空态
   尚无可展示的研究卡片时，提示返回输入页重新发起研究。
2. 加载态
   正在载入已生成卡片或等待卡片刷新时，显示骨架或明确加载反馈。
3. 成功态
   成功展示结构化研究卡片，并允许转化为项目。
4. 错误态
   卡片拉取失败或转化前校验失败时，显示错误摘要与返回/重试入口。

### `paper_project_detail_page`

1. 空态
   项目刚创建且尚未完善结构、素材或构建结果时，页面仍应显示可操作的初始工作区。
2. 加载态
   正在加载项目详情、章节或最近构建记录时，显示局部或整页加载反馈。
3. 成功态
   项目数据完整加载后，所有受控区域可见并可操作。
4. 错误态
   项目加载失败时，显示可见错误、trace 线索和返回入口，而不是空白页。

### `template_selection_panel`

1. 空态
   尚未选择模板时，明确显示“未选择模板”及下一步动作。
2. 加载态
   正在拉取模板目录或应用模板切换时，禁用重复切换。
3. 成功态
   展示当前模板、可切换选项和映射结果。
4. 错误态
   模板目录加载失败或映射不兼容时，显示明确错误与恢复路径。

### `asset_panel`

1. 空态
   尚无项目素材时，显示可添加素材的入口与格式说明。
2. 加载态
   正在读取素材列表或处理素材变更时，显示局部加载反馈。
3. 成功态
   正常展示素材列表及其基本操作。
4. 错误态
   素材读取或变更失败时，显示失败原因并允许重试。

### `build_status_panel`

1. 空态
   尚未触发任何构建时，显示“尚无构建记录”和首次构建入口。
2. 加载态
   构建排队或运行中时，显示 `queued` / `running` 的可见状态。
3. 成功态
   最近构建成功时，显示 `succeeded` 摘要并联动预览区域。
4. 错误态
   最近构建失败时，显示 `failed` 摘要、位置提示和修复建议。

### `pdf_preview_export_panel`

1. 空态
   尚无成功 PDF 产物时，明确显示暂无可预览文件。
2. 加载态
   正在载入预览文件或准备导出动作时，显示可见等待反馈。
3. 成功态
   成功显示最近一次 PDF 结果，并提供打开和导出操作。
4. 错误态
   预览文件缺失、加载失败或打开失败时，显示错误摘要并保留恢复路径。

## 5. 第 4 步范围外内容

以下内容会越界进入第 5 步或后续步骤，不得在本文档中冻结：

1. 研究输入支持的精确类型列表。
2. DOI、arXiv、标题或 URL 的检索优先级。
3. `research_card` 的字段 schema。
4. 转化为 `paper_project` 时的精确导入载荷。
5. 模板目录的最终条目与映射规则细节。
6. 章节编辑操作的完整规则集。
7. 构建触发后的 worker 协议细节。

## 6. 验证说明

第 4 步应依据以下检查项进行验证：

1. 从研究输入到 PDF 预览，完整用户旅程可仅凭本文档中的三个顶层页面和四个项目页区域走通。
2. 每个页面或区域都只有一个清晰主职责，并明确说明“不应做什么”。
3. 不存在为了“以后可能会用”而提前拆出的额外顶层路由。
4. 每个页面或区域都覆盖空态、加载态、成功态和错误态，且这些状态不依赖协作、在线同步或其他范围外能力。
5. 本文档未提前冻结第 5 步的研究输入流程、检索策略或研究卡片 schema。
