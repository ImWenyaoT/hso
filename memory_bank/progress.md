## 第一部分：基础产品契约

### 六步用户操作闭环（v1 冻结）

1. **输入研究起点** — 用户从关键词或支持的种子论文引用开始
2. **获取研究卡片** — 系统检索客观论文数据，返回结构化 `research_card`
3. **转化为论文项目** — 用户将选定结构化内容转化为 `paper_project`（核心对象被创建的时刻）
4. **选择模板** — 用户为 `paper_project` 绑定受支持的 `template` 包
5. **编辑论文结构** — 用户管理有序 `section` 列表及最小 `asset` 集合
6. **构建与预览** — 系统运行受控 LaTeX 构建，返回可预览结果

基础产品没有第七个隐藏操作。

### 核心对象词汇表

| 对象 | 定义 |
|------|------|
| `research input` | 用户提供的研究生成起点：关键词、DOI、arXiv ID、论文标题或论文 URL |
| `research card` | 结构化、客观、项目前置的研究产物。**不是**主工作对象，是上游输入材料 |
| `paper_project` | 基础产品的**核心工作对象**，持续承载模板、章节、素材、引用和构建历史 |
| `template` | 受控模板包，定义规范化内容如何映射到支持的输出格式 |
| `section` | `paper_project` 中有序的规范化章节单元（属于项目，不属于研究卡片） |
| `asset` | 附加到 `paper_project` 的构建相关文件（v1 为图像） |
| `build_job` | 一次 LaTeX 构建执行记录，绑定项目版本快照，状态为 `queued/running/succeeded/failed` |
| `build_result` | 一次 `build_job` 的已记录结果（产物路径、摘要、状态、时间戳），由 `build_job + build_artifact` 承载 |
| `reference_source` | 与项目关联的结构化引用来源记录，不是通用 asset |

### v1 非目标（明确排除，不得暗示部分支持）

1. 不支持协作工作流（v1 是本地单用户助手）
2. 不支持任意 LaTeX 项目导入
3. 不做主观学术判断（仅提供客观可溯源材料）
4. 不提供全功能编辑器或 LaTeX IDE
5. 不支持构建取消
6. 不支持用户自定义模板上传
7. 不提供应用商店级别的发布工作流
8. 不以润色或改写为主要工作流
9. 不提供独立于 `paper_project` 工作流之外的研究产品线
